*2013.01.17 사용자 뷰 만들기 
	DB로 부터 정보를 읽어 Master&Child 관계를 구한다. 
	전체 컬럼을 보여주고 선택한다. 
	각 컬럼의 이름은 사용자가 별도로 명명할 수 있다. 
	포멧도 담당자가 지정할 수 있다. (일자형, 금액, 주민번호 등)
	각 컬럼의 링크도 담당자가 지정한다. -> 자료 상세조회, 저장화면 등(?)
	각 회사별로 뷰를 정의할 수 있으며 Admin이 기본 컬럼정보를 조정한 후 
	각 담당자가 자기만의 뷰를 만든다. (데이타가 많아 질수 있겠다.)
	기본 뷰, 회사별 뷰, 담당자 뷰 3개가 만들어 진다. (흐미..)

	리스트 화면에 대하여 펼치기 기능을 제공한다. 
	각 뷰에서 기본화면보기와 펼쳐보기 기능을 제공하여 펼칠경우 전체화면으로 
	보여준다. -> 기본칼럼, 확장칼럼을 선택할 수 있어야 하며, 
	각 화면별 정렬조건, 머지 조건등을 설정할 수 있어야 한다. 

	특별한 화면의 경우 확장이 안된다. 평가 메트릭스, 트리뷰, 별도의 시뮬레이션 기능 등은
	확장기능을 부여할 수 없다. 
	조건과 뷰를 연결하는 작업은 JSP에서 처리한다. 컴포넌트 뷰로 구성이 된다. 
	추가 컬럼(합계, 평균, 카운트 등)의 경우 입력할 수 있어야 하며, 
	정렬순서, Link, 머지 등에 대해서는 리스트를 통째로 서버로 전달한다. 

http://cafe.naver.com/eclipseplugin.cafe

tomcat에서 한글이 깨지는 문제 
  - tomcat서버의 server.xml에 URIEncoding="UTF-8"을 추가한다. 
 변경 전 : <Connector connectionTimeout="20000" port="8080" protocol="HTTP/1.1" redirectPort="8443"/>
